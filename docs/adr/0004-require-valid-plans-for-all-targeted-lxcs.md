# Require valid plans for every targeted LXC

Full lifecycle execution crosses the lifecycle planning barrier only when every targeted LXC has a valid plan; any compilation, identity, required-observation, or unsatisfied intent precondition blocks actions for the entire targeted set. Lifecycle intent is a strict set of permitted transitions: if the requested outcome requires a transition outside that set, planning fails instead of silently skipping it. Configure-only lifecycle therefore requires every target to be running rather than silently skipping absent or stopped LXCs. When release-based rebuild policy is enabled, the guest release is a required observation and an unprobeable stopped target blocks planning; when that policy is disabled, release observation is not required. Every target still receives a semantic result when the barrier blocks execution, distinguishing the planning failure from otherwise valid plans that were not executed. This replaces partial-skip execution for planning failures. Standalone validation aggregates all detected problems before exiting nonzero when any planning failure exists, and command limits keep unrelated inventory hosts outside the barrier.
