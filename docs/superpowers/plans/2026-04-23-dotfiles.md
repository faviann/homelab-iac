# Dotfiles Repo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `faviann/dotfiles` private GitHub repo that manages personal dotfiles across laptop, WSL2, and LXC workstation using chezmoi + Bitwarden CLI, with secrets never touching the repo.

**Architecture:** The repo root is the chezmoi source directory. Machine type (fish machines vs LXC) is detected by hostname in `.chezmoi.toml.tmpl`. Secrets are fetched from Bitwarden at `chezmoi apply` time via `BW_SESSION` — nothing secret is committed. Fish config is skipped on the LXC via `.chezmoiignore`.

**Tech Stack:** chezmoi 2.x, Bitwarden CLI (`bw`), fish shell, bash, git

---

## Task 4: Create Bitwarden items (manual)

This task has no code. Create the items in Bitwarden before the next task.

- [ ] **Step 1: Log in to Bitwarden**

```bash

export BW_SESSION=$(bw unlock --raw)
```

- [ ] **Step 2: Create each secret as a Secure Note**

For each item below, run:
```bash
bw create item "$(bw get template item | \
  jq --arg name "$NAME" --arg notes "$VALUE" \
  '.type=2 | .name=$name | .notes=$notes | .secureNote.type=0')"
```

| `$NAME` | `$VALUE` |
|---|---|
| `dotfiles/anthropic-api-key` | your Anthropic API key |
| `dotfiles/openai-api-key` | your OpenAI/Codex API key |
| `dotfiles/copilot-token` | your GitHub Copilot token |
| `dotfiles/gh-token` | your GitHub personal access token |

Or create them via the Bitwarden web UI as Secure Notes with those exact names.

- [ ] **Step 3: Verify items are accessible**

```bash

```

Expected: prints the key value (do not paste or share this output).

function fish_greeting
    if test -z "$BW_SESSION"
        echo "⚠  Bitwarden locked. Run: export BW_SESSION=(bw unlock --raw)"
        return
    end

    if not bw unlock --check &>/dev/null

- [ ] **Step 1: Create `run_once_install-packages.sh.tmpl`**

```bash
#!/bin/bash
set -e

{{ if not .is_lxc -}}
if ! command -v fish &>/dev/null; then
    sudo apt update
    sudo apt install -y fish
fi
{{ end -}}
```

The `run_once_` prefix means chezmoi runs this script exactly once per machine (tracked by a hash). On subsequent `chezmoi apply` calls it is skipped unless the script content changes.

# Bootstrap — New Machine Setup

Run these steps in order on any new machine.

## 1. Generate SSH key (once per machine)

```bash
ssh-keygen -t ed25519 -C "faviann@gmail.com" -f ~/.ssh/id_ed25519
```

Then add `~/.ssh/id_ed25519.pub` to GitHub:
- **Settings → SSH and GPG keys → New SSH key**
- Add once as **Authentication Key**
- Add again as **Signing Key**

## 2. Install chezmoi and Bitwarden CLI

```bash
sh -c "$(curl -fsLS get.chezmoi.io)" -- -b ~/.local/bin
sudo snap install bw
```

## 3. Unlock Bitwarden

```bash
bw login                                  # first time only
export BW_SESSION=$(bw unlock --raw)
```

## 4. Apply dotfiles

```bash
chezmoi init --apply git@github.com:faviann/dotfiles.git
```

## Day-to-day: pull and re-apply

```bash
export BW_SESSION=$(bw unlock --raw)
chezmoi update
```

## Verify

```bash
cat ~/.gitconfig           # git config applied
cat ~/.ssh/allowed_signers # signing key line present
echo ${#ANTHROPIC_API_KEY} # > 0 means secret injected
fish_greeting              # should be silent if BW_SESSION is set
```

## Hostname contract

The LXC workstation must be named `workstation` (set by the Ansible repo).
This is what triggers `is_lxc = true` in `.chezmoi.toml.tmpl`, which skips
fish config on that machine.
```
